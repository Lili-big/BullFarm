# BullFarm 选股 Agent 核心说明

## 项目定位

BullFarm 是一个面向 A 股短线到小波段机会的规则型选股 Agent。它的目标不是替代投资判断，也不是给出无条件买入建议，而是把一套稳定的交易观察框架落实成每天可重复执行、可复盘、可追踪的选股工作流。

项目希望解决的问题是：每天市场里有大量股票在异动，但真正值得投入注意力的机会很少。BullFarm 要做的是先把噪声过滤掉，再把少数更可能具备 2-3 日延续性的标的整理成清晰的观察清单，并告诉使用者为什么它们值得看、哪里还不够、什么时候只能观察、什么时候必须回避。

## 想实现的核心效果

1. 每天形成一份有纪律的候选清单

   Agent 会围绕中期趋势、近 5 日启动、板块强度和大盘环境四个维度，筛出更接近“趋势健康、刚刚启动、板块有共振、市场允许参与”的股票。最终结果不是简单排名，而是把每只股票放入“强参与、轻仓试错、只观察、回避”等决策层级。

2. 把“能不能买”拆成更稳健的问题

   项目不鼓励看到涨幅或热点就直接参与，而是要求同时回答：

   - 这只股票是否还在健康趋势里？
   - 最近几天是否出现有效启动，而不是短线透支？
   - 所属方向是不是近期市场主线？
   - 当前大盘环境是否支持进攻？
   - 如果判断错了，什么条件触发止损或退出？

   只有这些条件同时较好时，Agent 才允许更积极的结论；如果大盘、板块或个股结构不配合，即使个股分数高，也会降级为观察或回避。

3. 输出可执行的观察与风控语言

   每次结果应让使用者快速知道：

   - 今日优先关注哪些股票；
   - 每只股票属于哪类机会；
   - 未来 2-3 日更可能强延续、弱延续、分歧转强，还是强转弱；
   - 合适的触发条件是什么；
   - 失效条件和止损线在哪里；
   - 当前是否受市场环境限制，需要降低参与级别。

   这份输出要服务于盘前准备、盘中观察和盘后复盘，而不是只给一个分数表。

4. 建立每日归档和历史复盘闭环

   项目希望每一次选股都能留下当天快照和结果记录，后续继续追踪 T0、T1、T2、T3、T5、T10 等时间窗口的表现。这样使用者可以回头判断：

   - 哪类结论真正有效；
   - “强参与”“轻仓试错”“只观察”“回避”之间是否拉开差异；
   - 哪些市场环境下策略更容易失效；
   - 规则需要强化、放宽，还是重新定义。

   也就是说，BullFarm 不只是每天选股，还要帮助验证这套选股框架是否真的有用。

5. 让结果可以被页面和数据服务持续展示

   项目最终希望把每日结果沉淀成一个可浏览的选股看板：既能看当天候选，也能看历史批次、策略表现、分层收益和复盘状态。这样使用者不需要在零散文件之间来回查找，而是可以从一个统一入口观察策略运行情况。

## Agent 的判断框架

BullFarm 的判断顺序是从“是否值得看”到“是否允许参与”逐层收紧。

第一层是中期趋势。优先寻找已经形成 60-120 日趋势基础、均线结构较健康、处于平台突破、趋势回踩或强势震荡再启动状态的股票。下降趋势里的反抽不作为主要机会。

第二层是近 5 日启动。关注低点是否抬高、短期均线是否向上、上涨是否有量能支持、回调是否缩量，以及是否出现高位长上影等强转弱信号。短期涨幅过度透支时，即使趋势仍好，也要谨慎或回避。

第三层是板块共振。个股最好处在近期市场主线中，而不是孤立上涨。板块需要相对大盘更强、成交活跃、有前排强势股带动，个股本身也应尽量是前排或有辨识度的标的。

第四层是大盘许可。大盘环境决定仓位和进攻程度。如果指数、成交、涨停溢价、亏钱效应都不支持进攻，Agent 应自动降低参与建议，避免把个股强度误判为可重仓机会。

## 典型使用场景

- 盘前：生成今日观察池，标出优先级和参与条件。
- 盘中：对照触发条件，判断某只股票是否从观察转为可试错。
- 收盘后：保存当天结果，作为后续收益追踪和策略复盘的基准。
- 周期复盘：比较不同结论层级、不同板块、不同市场环境下的实际表现。
- 策略修正：根据复盘结果调整筛选权重、风险边界和降级条件。

## 输出应该长什么样

理想输出不是“推荐买入列表”，而是一份结构化的交易观察报告。它应该包含：

- 今日市场环境摘要；
- 候选股票排名和决策层级；
- 每只股票的机会类型；
- 主要看多理由；
- 主要风险或缺口；
- 买点触发条件；
- 止损或退出条件；
- 是否因板块或大盘环境被降级；
- 本次结果后续复盘所需的标记。

如果市场环境不好，最好的结果可能是一份高质量的“只观察”和“回避”清单，而不是强行给出买入机会。

## 风险边界

BullFarm 必须始终保持以下边界：

- 不把规则筛选结果包装成确定性投资建议；
- 不因为单日涨停、短线大涨或热点概念就放弃趋势和风控检查；
- 不在大盘环境明显偏弱时给出激进参与结论；
- 不忽略流动性、ST、强转弱、长上影、短期透支等硬性风险；
- 不用未经确认的实时行情或缺失数据得出强结论；
- 不追求每天都有可参与标的，空仓或只观察也是有效输出。

## 项目成功标准

这个项目真正成功，不是某一天选出一只上涨股票，而是形成一套稳定、可重复、可检验的选股辅助系统。

成功状态应该是：

- 每个交易日都能生成清楚的候选清单；
- 每个结论都有可解释的理由和风险边界；
- 每次结果都能归档并用于后续复盘；
- 历史数据能说明哪些规则有效、哪些规则需要调整；
- 使用者能通过报告或看板快速理解“今天该看什么、为什么看、什么时候不该动”；
- Agent 的输出能帮助降低情绪化追涨和随意交易，而不是制造更多冲动。

## 最终愿景

BullFarm 希望成为一个“纪律化交易观察助手”：每天稳定地把市场机会压缩成少数值得跟踪的对象，把参与条件和风险条件说清楚，再通过持续复盘验证策略本身。

它不追求神奇预测，而追求一致性、可解释性和可复盘。长期来看，项目要帮助使用者建立一套更冷静、更可验证的短线观察体系。
